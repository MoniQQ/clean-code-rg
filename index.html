Reading Group

Clean Code: A Handbook of Agile Software Craftsmanship

By Robert C Martin / @unclebobmartin

Robert C. Martin

  • software engineer, co-author of the agile manifesto
  • frequent speaker at conferences
  • advocates for software craftsmanship, SOLID principles, good OO design
  • and incremental changes, TDD, all that good stuff

Why?

"[Most managers] may defend the schedule and requirements with passion; but that's their job. It's your job to defend the code with equal passion."

Readers will come away from this book understanding:

  • How to tell the difference between good and bad code
  • How to write good code and how to transform bad code into good code
  • How to create good names, good functions, good objects, and good classes
  • How to format code for maximum readability
  • How to implement complete error handling without obscuring code logic
  • How to unit test and practice test-driven development

Reading/learning is hard

The book has 400 pages.

At 1 minute/page you need 6.66 hours for reading alone.

And reading isn't much without practice.

The plan

We will...

  • Read 1-3 chapters a week (30 - 50 pages, less than 1h)
  • Meet weekly for 1 hour
  • Focus on how we can apply what we read, using real code or katas
    • ... so bring relevant code/questions to the meetings
